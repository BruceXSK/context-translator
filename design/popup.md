# popup

插件弹窗：目标语言快速切换 + 打开设置页入口、当前页 token 用量展示、压缩/清空 session 动作。完整设置（端点 / API key / 模型 / 触发键 / 自定义提示词）在独立设置页（POP-006）；手动补充指引（Add instruction）移至右键无选区入口（POP-002）。

## Specs

- POP-001 [DONE] popup quick settings: the popup carries only the target-language selector (default zh-CN) and an entry to open the full settings page; it persists targetLang to chrome.storage.local. (Scope narrowed from the full settings form — base URL/API key/model/trigger key moved to the options page, which also hosts the custom prompt; see POP-006. The system prompt is now a non-editable code constant, not a user setting — see ARCH-013.)
- POP-002 [DONE] Add instruction input: a right-click "Add instruction" menu entry (shown only when no text is selected, BG-003) opens an inline input panel (shadow-DOM, textarea + submit; Enter to submit, Shift+Enter for newline, Esc to close) where the user types supplementary guidance (domain/terminology/tone); on submit the text is added to the active page's pending-context buffer as an instruction (session.addInstruction → SES-002 instruction kind, folded into the next translation's <user-instruction> block). Re-scoped from the popup text box: the no-selection right-click entry is a more discoverable surface for typed instructions and shares the page's context (the popup cannot read the page).
- POP-003 [DONE] token usage display: on open the popup queries the active tab's content via `getUsage` (which returns both cumulative and last-translation usage, SES-005) and renders three rows. Context usage = the latest translation response's `prompt_tokens` against `maxContextK*1000` (CFG-006), shown as `{usedK}K/{maxLabel} ({pct}%)` with a progress bar whose used portion is gray; `usedK` and `pct` are integers (rounded); `maxLabel` is `M` when `maxContextK`≥1000 else `K` (1000→1M). Two cache hit rates, each `hit/(hit+miss)` as a percentage with one decimal: Last (latest translation response) and Total (`Σhit/(Σhit+Σmiss)` across responses that reported cache fields). Unavailable fields show `—`: no translation yet (no last usage), non-DeepSeek responses (no cache fields), or the active tab has no content script (restricted pages). The first translation on a page shows Last 0.0% (cache building) and the rate climbs with turns; after a compress the context gauge holds at the last translation's value until the next translation, then drops (history→summary) — expected per ARCH-006/DS-001.
- POP-004 [PLAN] compress context action: a button triggers session compression on the active tab.
- POP-005 [PLAN] clear session action: a button resets the active tab's session.
- POP-006 [DONE] options page: a standalone settings page (open_in_tab) hosts base URL, API key, model, a max context field (number, unit K, default 1000 = 1M for deepseek-v4-flash; the model's context window, user-supplied since the API doesn't return it — denominator for the popup context gauge, CFG-006/POP-003; a non-positive value falls back to 1000K), trigger key (a single-select of Alt/Shift/Ctrl, default Alt; stored as the standard KeyboardEvent.key name so the content-script key matcher hits), a custom prompt (user-supplementary guidance, default empty — see CFG-005; folded into the first user message's <user-instruction> block, supplements but cannot override the non-editable system prompt, ARCH-013), a thinking toggle (switch style, default off; turning it on shows a notice toast warning that translations may stay on "Translating…" longer while the model thinks), and an effort single-select (segmented capsule, Low/Medium/High/Max, default Low, stored lowercase); when thinking is off the effort select is disabled and greyed. It reads/writes the same chrome.storage.local via the config module (shared schema with the popup), so changes take effect browser-wide — immediate for baseUrl/apiKey/model/thinking/effort (the stateless service worker reads them per request), while trigger key and custom prompt apply to pages loaded after the change (the content script snapshots them on load).

## Layout

- 设置区：目标语言（POP-001）+ 打开设置页入口。
- 当前页区：Context 用量（最后翻译 `prompt_tokens` / `maxContextK`，整数 K + 整数 % + 灰色进度条）、Last/Total 缓存命中率（`hit/(hit+miss)`，一位小数 %）（POP-003）；「压缩上下文」「清空 session」按钮（POP-004 / POP-005）。手动补充指引（Add instruction）移至右键无选区入口（POP-002），不在 popup。
- 设置页（POP-006，独立 open_in_tab 页）：base URL / API key / model / 触发键 / 自定义提示词（custom prompt，用户补充指引，默认空）/ 思考模式（开关，默认关；开启时弹通知提示翻译可能变慢）/ 思考强度（Low/Medium/High/Max 单选，默认 Low；思考模式关时禁用灰化）。

## Communication

通过 `chrome.tabs.query` 取当前活动 tab，经 `chrome.tabs.sendMessage` 与该页 content 通信（读取用量、触发压缩/清空）。目标语言写入 config（`chrome.storage.local`）。设置页（POP-006）直接读写 config 的 `chrome.storage.local`，不经 content；「打开设置」入口调 `chrome.runtime.openOptionsPage()`。手动补充指引（Add instruction）现由右键菜单 → content 内联输入面板完成（POP-002），不经 popup。
