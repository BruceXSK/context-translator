- CT-009 (2026-06-25): modified — retry control scoped to the selection floating panel; the embedded hover block no longer shows a retry button and retries by re-pressing the trigger key.

Timer-driven (not CSS) because the block is light-DOM with no shadow (CT-010). (Revised from dot-count cycling to a braille spinner for a livelier, less monotonous feel.)
- POP-001 (2026-06-25): modified — popup scope narrowed from the full settings form to the target-language selector plus an entry to open the settings page; base URL/API key/model/trigger key/system prompt moved to the options page (POP-006).
- POP-006 (2026-06-25): added — standalone options page (open_in_tab) hosting base URL/API key/model/trigger key/system prompt. Trigger key is a single-select of Alt/Shift/Ctrl (default Alt), stored as the standard KeyboardEvent.key name (Alt/Shift/Control) so the content-script key matcher (e.key) hits; reads/writes the same chrome.storage.local via the config module (shared schema with the popup); changes take effect browser-wide — immediate for endpoint/key/model (the stateless service worker reads them per request), trigger key and system prompt apply to pages loaded after the change.
